## 摘要
智能体系统通过调用外部工具和维持持久化内存来增强其能力。然而，这些外部依赖引入了新的攻击面。最近的工具和内存投毒攻击表明，恶意构造的工具描述符和中毒内存可以隐蔽地偏置智能体行为。这些威胁反映了一个更深层次的问题：智能体用于规划和执行的上下文状态缺乏可验证的连续性。我们提出ElephantAgent，一种强制执行上下文状态连续性以防御上下文状态投毒的攻击。受先前状态连续性机制（如Nimble）的启发，ElephantAgent将这种保护扩展到智能体系统不断演化的上下文状态。我们将上下文状态定义为智能体整个上下文中有限的、安全关键的子集（例如，工具状态和内存）。在处理每个查询之前，ElephantAgent重新计算本地上下文状态的摘要，并对照最新的授权摘要进行验证。通过使用复制的可信硬件，ElephantAgent维护一个授权的上下文状态转换的线性化账本，并检测带外状态篡改。为了处理带内语义滥用，ElephantAgent额外提供历史可追溯性，支持条件性事后审计和恢复到已知良好的先前状态。

## Abstract
Agentic systems enhance their capabilities by invoking external tools and maintaining persistent memory. However, these external dependencies introduce novel attack surfaces. Recent tool and memory poisoning attacks show that maliciously crafted tool descriptors and poisoned memory can covertly bias agent behavior. These threats reflect a deeper issue: the lack of verifiable continuity in the agent's contextual state for planning and execution. We present ElephantAgent, a protocol that enforces Contextual State Continuity to defend against contextual state poisoning. Inspired by prior state-continuity mechanisms (e.g., Nimble), ElephantAgent extends this protection to the evolving contextual state of agentic systems. We define the contextual state as the bounded, security-critical subset of the agent's entire context (e.g., tool state and memory). Before processing each query, ElephantAgent recomputes the digest of the local contextual state and verifies it against the latest authorized digest. Using replicated trusted hardware, ElephantAgent maintains a linearizable ledger of authorized contextual state transitions and detects out-of-band state tampering. To handle in-band semantic abuse, ElephantAgent additionally provides Historical Traceability, enabling conditional post-hoc audit and recovery to a known-good prior state.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：受传统状态连续性机制（如Nimble[3]）启发，将状态连续性保护扩展到智能体系统的上下文状态。方案由两个新增组件组成：
  - **Context Guard (CG)** ：运行在可信执行环境（TEE）内的主机侧组件，负责在每次查询前重新计算本地上下文状态摘要，并与SCM授权的最新摘要进行验证；状态更新采用“先提交再最终化（commit-before-finalize）”流程。
  - **State Continuity Module (SCM)** ：部署为复制的TEE后端服务（基于Nimble的线性化账本），维护一个只追加、可验证的授权上下文状态摘要账本。每个账本记录有序的序列ID和摘要，并签发签名收据。
- **关键技术细节**：
  - **上下文状态定义**：\( C_t = \langle T_t, M_t \rangle \)，其中工具状态\( T_t \)为已注册的工具描述符，记忆\( M_t \)包括长期指令、查询级事件转录（用户查询+代理响应）和托管工件。
  - **状态初始化**：通过ApprovePolicy（原型中使用Trust On First Use）批准初始状态，计算摘要并在SCM中锚定，同时密封快照。
  - **状态验证（Algorithm 3）**：CG生成新鲜随机数\( n_r \)，调用SCM.GetLatestState获取带签名的状态证明，验证签名和随机数，并检查本地摘要是否匹配最新授权摘要。失败则中止查询（fail‑closed）。
  - **状态更新（Algorithm 4）**：CG执行预览确定下一摘要，调用SCM.Commit提交新摘要和递增序列ID，验证收据后密封待更新记录，最终化本地副作用，最后将记录追加到本地账本并密封。
  - **安全分析**：在标准密码学假设（SCM账本正确性、签名不可伪造、哈希抗碰撞、CG可信且不可绕过、认证与完整性保护）下，证明了上下文状态连续性和历史可追溯性。

## 6. 主要结论与发现

- **性能开销极低**：ElephantAgent引入的端到端开销仅1.02×–1.04×（相对于NoGuardAgent）。其中初始化开销（8.22×）是一次性成本；每查询的验证和更新开销分别约4.2–4.3 ms和5.2–5.4 ms。
- **上下文状态连续性可以有效检测带外篡改**：在给定密码学假设下，任何对受保护上下文状态的未授权修改都会导致VerifyState失败，从而阻止代理规划和执行。
- **历史可追溯性提供事后审计与恢复能力**：本地账本和快照可用于识别首次恶意状态转换，并可能恢复到已知良好先前状态（需人工审计）。
- **消融实验表明MCP Server使用TEE有额外但可接受的开销**：初始化时间增加1.51×，工具调用时间增加1.12×–1.22×。

## 7. 优点

- **方法创新性**：首次将状态连续性机制系统性地应用于智能体系统，形式化了上下文状态连续性和历史可追溯性两种安全属性。
- **防御确定性**：基于密码学验证而非统计检测，避免了误报/漏报问题，安全分析具有严格形式化证明。
- **协议通用性**：作为附加安全协议，可应用于MCP和CLI等多种代理框架，不依赖特定LLM或tool set。
- **实验设计合理**：通过主动选择低延迟模型和轻量任务，突出显示了防御本身的开销，避免了LLM处理时间掩盖测量结果。
