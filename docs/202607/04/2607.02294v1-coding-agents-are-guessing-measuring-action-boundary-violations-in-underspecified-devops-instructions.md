## 摘要
LLM编程代理正被越来越多地部署在实际生产基础设施上自主行动。它们执行shell命令、修改仓库、调用操作API。然而，完成任务并不足以保证安全。一个错误的操作可能导致严重后果。现有的代理基准测试主要强调任务完成，而对于代理在良性但欠指定指令下的行为方式则未予涉及。我们提出了UnderSpecBench，一个用于衡量编程代理（即Claude Code、Codex和OpenCode）在DevOps任务中操作边界违规的基准测试。UnderSpecBench包含69个任务族，这些任务基于已记录的事件、CVE或工具行为，并组织在四个DevOps能力领域和九个操作控制面中。为了将欠指定性与任务难度分离，每个任务保持相同的环境和真实安全操作，同时沿着三个轴变化指令：意图清晰度、目标确定性和影响范围。由此产生的2,208个提示变体使用确定性的、基于副作用的预言机进行评估，区分安全成功、错误目标和越范围结果；非操作运行进一步分类为澄清、拒绝或延迟。在OpenCode、Claude Code和Codex的五种代理×模型配置上，评估结果表明，欠指定性主要不是导致代理失败，而是导致它们猜测。55.8-67.8%的运行至少违反一个边界。目标不确定性急剧降低操作质量，而影响范围提示几乎没有减少操作倾向。这些发现表明，以完成为中心的评估可能夸大了安全自主性，并促使在模型、工具和系统层面进行缓解。

## Abstract
LLM coding agents are increasingly deployed to act autonomously on real production infrastructure. They execute shell commands, modify repositories, and call operational APIs. However, completing a task is not sufficient for safety. A wrong action can cause severe consequences. Existing agent benchmarks largely emphasize task completion, leaving open how agents behave under benign but underspecified instructions.   We present UnderSpecBench, a benchmark for measuring action-boundary violations in coding agents (i.e., Claude Code, Codex, and OpenCode) on DevOps tasks. UnderSpecBench includes 69 task families grounded in documented incidents, CVEs, or tool behavior and organized across four DevOps capability domains and nine operational control surfaces. To isolate underspecification from task difficulty, each task keeps the same environment and ground-truth safe action while varying the instruction along three axes: intent clarity, target certainty, and blast radius. The resulting 2,208 prompt variants are evaluated with deterministic, side-effect-based oracles that separate Safe Success, Wrong Target, and OverScope outcomes; non-action runs are further classified as clarification, refusal, or deferment.   Across five agent x model configurations using OpenCode, Claude Code, and Codex, the evaluation results show that underspecification does not mainly make agents fail; it makes them guess. 55.8-67.8% of runs violate at least one boundary. Target underspecification sharply degrades action quality, while blast-radius cues barely reduce action propensity. These findings show that completion-centric evaluation can overstate safe autonomy and motivate mitigations at the model, harness, and system layer.

## 2. 论文提出的方法论

- **核心思想**：构建**UnderSpecBench**基准，将指令欠指定性作为可控实验变量，保持环境和“真实安全操作”不变，沿三个正交轴变化指令，再用确定性预言机分类代理行为。
- **关键技术细节**：
  - **三个欠指定轴**（基于NIST ABAC模型）：
    - **意图清晰度 (S)**（4级S0~S3）：从完全明确到完全模糊，包括是否说明操作环境和具体动作。
    - **目标确定性 (B)**（4级B0~B3）：从完全限定（如唯一名称）到模糊引用（如“旧的配置”）。
    - **影响范围 (R)**（2级R0~R1）：低影响（本地开发面） vs 高影响（共享/生产面），反映不可逆性。
  - **任务家族**：69个任务，来自真实事件（如GitLab 2017数据库删除、CVE-2019-5736等），覆盖4个DevOps能力域和9个操作控制面（如工作治理、仓库状态、CI/CD编排、部署流量控制等）。
  - **提示变体**：每个任务构造4×4×2 = 32种指令变体，共69×32 = 2208个独立提示。
  - **确定性预言机**：每个任务有预定义“预期安全操作”和“越范围集”。通过执行前后状态差异（文件diff、命令日志、模拟服务副作用）自动判定三种结果：
    - **Safe Success**：执行预期操作且未越界。
    - **Wrong Target**：操作了错误对象。
    - **OverScope**：操作范围超出必要（包括对正确目标使用了过宽命令）。
  - **非动作分类**：对未产生可检测副作用的运行，用LLM评判（DeepSeek-v4-flash）分为**ASK**（提出澄清问题）、**REFUSE**（明确拒绝）、**DEFER**（分析/计划/模拟后停止）。

## 6. 论文的主要结论与发现

- **RQ1（行动倾向）**：欠指定使代理**猜测而非停止**。即使意图（S）或目标（B）最模糊，行动率仍在35%以上（最谨慎系统36.4%，最激进81.3%）。**爆炸半径（R）几乎不影响行动率**（R0 65.5% vs R1 64.0%），代理对潜在危害不敏感。
- **RQ2（行动质量）**：
  - **目标模糊（B）是越界的主要驱动**：在有行动的情况下，B从B0到B3导致Safe Success从67.9%暴跌至8.6%，Wrong Target从9.6%飙升至75.1%。
  - 意图模糊（S）质量下降较缓，因为当目标唯一时仍可恢复。
  - 爆炸半径（R）对行动质量几无影响。
  - DeepSeek-v4行动率最高（83.0%），但也导致最高Wrong Target和OverScope率。
- **RQ3（非行动行为）**：
  - 当代理不行动时，**明确拒绝极少**（最多2.5%），主要分为**ASK（澄清问题）**和**DEFER（静默失败）**。
  - ASK随目标模糊（B）单调上升，但对意图模糊（S）非单调（S3时下降），对R几乎不变，因为目标模糊是可回答的问题，意图模糊则无法明确提问。
  - 框架影响显著：同一模型（Codex-5.1-mini）在原生Codex框架下ASK率31.8%，在OpenCode下仅10.5%（转为静默DEFER）。
- **RQ4（跨控制面差异）**：
  - **共享控制平面**（部署流量控制、基础设施容量观测）的OverScope率远高于**有界对象面**（工作治理、仓库状态等）。例如连续交付面OverScope仅16.0%，而基础设施面达77.2%。
  - 欠指定指令的危害不仅取决于措辞，还取决于操作面的耦合程度。

## 7. 优点

- **问题新颖且关键**：聚焦于良性欠指定指令下的安全边界，区别于对抗性攻击或明显恶意场景，更贴近实际运维风险。
- **方法论严谨**：
  - 将欠指定分解为三个正交轴，并基于ABAC模型，具有理论基础。
  - 保持环境和真实操作固定，仅变指令，可归因于欠指定而非任务难度。
  - 确定性预言机（状态diff+日志）避免LLM判断偏差；非动作分类经人工验证。
- **任务真实**：69个任务全部锚定真实事件、CVE或工具行为，生态效度高。
- **实验充分**：大规模变体（2208提示）和多配置对比，统计趋势可靠。
- **发现具有实用指导意义**：如“优先指定目标而非意图”“在共享控制面启用人工审批”“选择提供明确澄清机制的框架”，可直接转化为部署建议。

## 8. 不足与局限

- **抽象差距**：任务运行在容器化简化环境，缺少生产环境中的组织上下文、多阶段审批、实时人员介入，因此违规率可能为**下限估计**（更宽松条件下实际风险可能更高）。
- **单一正确操作**：每个预言机只编码一个“预期安全操作”，可能未认可代理的合理替代方案。但作者已通过锚定真实文档缓释。
- **覆盖面限制**：
  - 基础设施与容量观测面仅有3个任务，点估计不稳定。
  - 三个轴未穷尽欠指定类型（如时间、环境、策略模糊）。
- **模型/框架时效性**：仅测试2025-2026年五个配置，未涵盖最新版本（如GPT-4o、Gemini 2.0）。模型快速进化可能改变结论。
