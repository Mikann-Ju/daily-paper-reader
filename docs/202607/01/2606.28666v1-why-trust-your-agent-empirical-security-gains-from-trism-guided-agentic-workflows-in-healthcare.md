## 摘要
基于智能体的AI通过将应用程序工具和资源暴露给大语言模型（LLMs）实现了任务自动化。然而，为了提高覆盖范围和准确性，智能体通常被授予超越普通用户的访问权限，从而引入了显著的安全风险。AI在集成到应用程序时往往忽视安全性，导致数据泄露和法规违规风险。本文应用AI信任、风险与安全管理（TRiSM）框架于医疗报告生成应用，展示了如何将不安全的智能体工作流转变为具有安全意识的智能体工作流。两种工作流在五种LLMs（Claude Haiku 4.5、GPT-4.1-nano、GPT-4.1-mini、GPT-5.4-mini和Gemini 2.5 Flash）上对两种报告类型进行了评估，总计800次生成和500个攻击场景，包括RAG投毒、数据字段注入和客户端网络注入。TRiSM引导的智能体工作流将RAG投毒的平均攻击成功率从31%降至10%，数据字段注入从42%降至25%，并通过服务器端提示构建完全消除了网络注入向量。此外，智能体工作流将报告准确性提高了14个百分点（从72.5%提升至86.5%），展示了能够提供更可靠输出的安全设计。本文通过展示最低权限、纵深防御的智能体工作流在提升安全性和准确性方面的效果，为知识做出了贡献，同时强调了模型选择是必要的架构考量。

## Abstract
Agent-based AI has enabled the automation of tasks by exposing application tools and resources to large language models (LLMs). However, to improve scope and accuracy, agents are often given access rights that exceed those of ordinary users, introducing significant security risks. AI is routinely integrated into applications with a disregard to security, risking data exposure and breaching regulations. This paper applies the AI Trust, Risk, and Security Management (TRiSM) framework to a medical report-generation application to demonstrate how an insecure agent workflow can be transformed into security-conscious agentic workflow. Both workflows were evaluated across five LLMs (Claude Haiku 4.5, GPT-4.1-nano, GPT-4.1-mini, GPT-5.4-mini, and Gemini 2.5 Flash) on two report types, totalling 800 generations and 500 attack scenarios including RAG poisoning, data-field injection, and client-side network injection. The TRiSM-guided agentic workflow reduced mean attack success rates from 31% to 10% for RAG poisoning and from 42% to 25% for data-field injection, while eliminating the network injection vector entirely through server-side prompt construction. Furthermore, report accuracy increased by 14 percentage points (72.5% to 86.5%) with the agentic workflow, demonstrating a secure design which provides more reliable outputs. This paper contributes to knowledge by demonstrating least-privilege, defence in depth agentic workflows improving security and accuracy, while also highlighting model choice is a necessary architectural consideration.

- **研究动机**：基于LLM的智能体（Agents）在医疗等高风险领域被广泛部署，但由于智能体常被授予超越普通用户的访问权限（如直接访问数据库、外部工具、MCP服务器等），引入了严重的安全漏洞，包括数据泄露、提示注入、RAG投毒、工具滥用等。现有研究多聚焦于单一攻击的防御，缺乏将安全框架系统性地应用于实际医疗工作流的实证研究。
- **整体含义**：本文以AI信任、风险与安全管理（AI Trust, Risk, and Security Management, TRiSM）五原则（治理、可信性、公平性、可靠性、数据保护）为指导，将不安全的多工具单智能体工作流重构为多智能体、最小权限、纵深防御的智能体工作流（Agentic Workflow），通过实证证明安全性大幅提升的同时，报告生成准确率也显著提高，并强调模型选择是安全架构的关键组成部分。

## 2. 方法论

### 核心思想
将原本一个“全能”智能体（拥有所有工具和资源访问权限）拆分为多个专门化子智能体，每个子智能体只拥有完成自身子任务所需的最小权限，以此实现数据隔离、工具隔离和攻击面缩减。同时将提示构建从客户端迁移到服务器端，消除客户端的网络注入风险。

### 关键技术细节
- **多智能体分解**：
  - **Patient Search Agent**：仅检索患者记录，输出JSON供下游使用，限制敏感数据暴露。
  - **Document Generation Agent**：生成医疗报告PDF，访问MCP服务器中的报告模板。
  - **Similar X-ray Agent**：针对新冠报告，检索X射线知识库并进行新冠预测，减少对生成式AI的依赖。
- **TRiSM映射**：
  - **数据保护**：服务端构建提示，执行最小权限工具分配，角色访问控制。
  - **公平性**：对输入进行嵌入分析（Embedding Analysis），识别已知注入模式，预处理后RAG信息。
  - **可靠性**：用户输入存储于数据库且禁止客户端写操作，避免聊天历史篡改。
  - **治理**：对每个智能体进行版本管理，使用单次调用（非对话式）以减少记忆中毒风险。
  - **可信性**：通过日志记录每个智能体调用的服务、工具和结果，实现可解释性。
- **服务端提示构建**：所有LLM请求在应用服务器端构建，客户端无法修改请求内容，从而完全消除客户端网络注入攻击向量。
- **策略JSON（Policy JSONs）**：定义每个智能体的权限边界和工具列表。

### 公式与算法流程（文字说明）
- 不安全工作流：单智能体 → 接收所有数据 → 直接调用LLM（客户端构建请求） → 输出报告。
- 安全智能体工作流：路由器 → 分解任务 → 子智能体1（仅患者信息）→ 子智能体2（仅报告模板+摘要）→ 子智能体3（仅X射线相似性） → 服务端汇总 → 输出报告。
- 注入检测：使用嵌入分析（embedding analysis）检测已知注入模式，若发现则阻止请求发送给LLM。

## 3. 实验设计

- **场景**：医疗报告生成，包括两种报告类型：常规住院报告 和 新冠住院报告。
- **数据集**：合成患者数据（未公开），应用内包含RAG知识库、X射线知识库、MCP工具等。
- **Benchmark**：将不安全单智能体工作流（Baseline）与TRiSM引导的多智能体工作流对比。攻击向量包括：
  1. RAG投毒（RAG Poisoning）：注入恶意内容到知识库。
  2. 数据字段注入（Data-field Injection）：在合法数据字段中嵌入攻击指令。
  3. 客户端网络注入（Client-side Network Injection）：拦截客户端请求并修改。
- **对比的方法/模型**：五种LLM：Claude Haiku 4.5、GPT-4.1-nano、GPT-4.1-mini、GPT-5.4-mini、Gemini 2.5 Flash。每种模型两种工作流，每种报告类型40次生成，共800次生成；攻击场景：每种向量20个攻击，共60个/模型/工作流，总计500次攻击（因为网络注入对智能体工作流不可行）。

## 6. 主要结论与发现

1. **安全性大幅提升**：
   - RAG投毒ASR从31%降至10%（p<0.0001）。
   - 数据字段注入ASR从42%降至25%（p=0.0009）。
   - 客户端网络注入通过服务端构建完全消除（结构性预防，非模型依赖）。
2. **准确性同步提高**：
   - 报告准确率从72.5%提升至86.5%（+14pp），所有模型均有提升，新冠报告提升尤其明显（GPT-5.4-mini提高27.5pp）。
3. **模型选择不是安全替代方案**：没有模型在所有向量上均达到抵抗，安全设计优于依赖模型。
4. **成本与性能**：常规报告成本增加不超过16%，但新冠报告中小模型（GPT-4.1-nano）成本增加51%且准确率低，说明复杂任务需选合适模型。
5. **双向错误传播**：智能体间交互偶尔导致新攻击成功（5/200场景），表明仍需在边界进行预处理和后处理检测。
