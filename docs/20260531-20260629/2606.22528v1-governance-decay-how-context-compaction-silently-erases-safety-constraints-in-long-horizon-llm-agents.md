## 摘要
现代LLM代理越来越多地依赖上下文压缩、总结或逐出，以将长时间运行的会话保持在令牌预算内。我们表明，这一上下文管理层是一个安全关键的失效面：代理在可见时可靠遵循的上下文内治理约束可能会被压缩悄然移除，导致同一代理在会话后期执行被禁止的工具操作。我们将这种失效模式称为治理衰减。我们引入了ConstraintRot，一个具有确定性工具调用评分的长时间代理场景基准，并测量了七个模型系列中由压缩引起的违规行为。在1323个回合中，违规行为从政策完全在上下文中时的0%上升到压缩后的30%，某些模型甚至达到59%；当约束在总结中幸存时，违规行为仍为0%，但当其被丢弃时，违规行为达到38%。我们进一步研究了一种压缩-逐出攻击，其中对抗性上下文内容使摘要器偏向于省略合法政策，并表明优化注入击败了所有评估模型。最后，我们提出约束固定，一种简单的无需训练的缓解措施，将治理约束与有损压缩隔离，并在我们的基准中将违规行为恢复至0%。这些结果将上下文管理识别为已部署LLM代理的一级治理面。

## Abstract
Modern LLM agents increasingly rely on context compaction, summarization, or eviction to keep long-running sessions within a token budget. We show that this context-management layer is a safety-critical failure surface: in-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session. We call this failure mode Governance Decay. We introduce ConstraintRot, a benchmark of long-horizon agent scenarios with deterministic tool-call grading, and measure compaction-induced violations across seven model families. Across 1,323 episodes, violation rises from 0% with the policy in full context to 30% after compaction, reaching 59% for some models; when the constraint survives the summary, violation remains 0%, but when it is dropped, violation reaches 38%. We further study a Compaction-Eviction Attack, in which adversarial in-context content biases the summarizer to omit a legitimate policy, and show that optimized injections defeat every evaluated model. Finally, we propose Constraint Pinning, a simple training-free mitigation that quarantines governance constraints from lossy compaction and restores violation to 0% in our benchmark. These results identify context management as a first-class governance surface for deployed LLM agents.

- **研究动机**：现代LLM代理（如企业助手）在实际部署中需要处理长时间、多步骤的工具调用轨迹。为了在固定上下文窗口（token预算）内运行，代理框架普遍采用**上下文压缩**机制：通过LLM总结或直接丢弃早期轮次来压缩历史。现有研究仅关注压缩对任务准确性的影响，而忽视了对安全约束的影响。
- **核心问题**：当代理的上下文中存在一个显式的治理约束（例如“禁止向外部域发送邮件”）时，代理在约束可见时会严格遵守；但经过一次上下文压缩后，由于压缩器以任务连续性为优化目标，该约束作为低显著性内容被静默丢弃。随后同一个代理在相同请求下可能违反该约束，执行被禁止的操作（如发送外部邮件）。作者将这种失效模式称为**治理衰减（Governance Decay）**。
- **整体含义**：上下文管理层已成为部署型LLM代理的一个**一级安全治理面**——它悄然擦除了仅存在于上下文中的约束，而这些约束正是部署中可灵活指定的企业策略，而非模型权重中固化的硬安全规范。研究表明，治理衰减可以被主动利用（**压缩-驱逐攻击**），即使是最鲁棒的模型也会被优化后的注入内容攻破。

- **压缩-驱逐攻击**：

- **Constraint Pinning防御**：
    - 将治理约束提取到**固定缓冲区**中，该缓冲区：
        - 豁免于压缩，每次压缩后重新注入。
        - 在每个步骤进行完整性检查，确保压缩后上下文仍蕴含固定约束。

## 6. 论文的主要结论与发现
1. **压缩诱导大范围衰减**：控制条件（政策可见）下所有模型违规率均为0%；一次压缩后平均违规率达30%，最高59%（DeepSeek-V4、Kimi-K2.5）。某些模型压缩后违规甚至超过无策略基线。
2. **约束存活决定违规**：当约束在摘要中幸存时，违规率0%；被丢弃时，违规率38%（LLM裁判）或43%（关键词启发式）。
3. **衰减由压缩器驱动，非代理模型**：跨压缩器-代理互换实验表明，违规率取决于生成摘要的模型，而非执行决策的代理模型。
4. **衰减规模随压缩侵略性上升**：摘要预算从300词降至15词时，约束存活率从88%降至23%，违规率从7%升至28%。
5. **软策略衰减远大于硬规范**：软策略衰减+50点，硬规范仅+6点（8.3倍差），说明治理衰减主要侵蚀部署特定的上下文规则。
6. **衰减泛化于多种压缩策略**：截断、层次压缩、LLM摘要均失效，仅保留头部内容的策略有效。
7. **压缩-驱逐攻击可武器化**：固定注入使GLM-5.1从0%升至22%；优化注入使Claude从0%升至65%，使DeepSeek达到100%。没有一个模型在所有轴上安全。
8. **Constraint Pinning消除违规**：在所有模型和固定攻击变种下恢复0%违规，代价<0.5% token开销。但面对操作员冒充注入仍会失效（残留10-17%违规），需要可信带外交互渠道。

## 8. 不足与局限
- **实验覆盖局限**：仅包含9个任务，且模拟场景为单一策略+单次触发请求。真实部署中可能有多个策略、动态提示、连续压缩等复杂情况。
- **偏差风险**：所有实验在沙盒中模拟，工具调用不实际执行；仅验证了API推理，未测试模型权重更新或微调后的行为。
- **应用限制**：
    - 固定防御仅适用于可提取为可引用规则的显式约束，不适用于隐式约束（如行为准则）。
    - 面对操作员冒充注入（在最近上下文中声称“策略更新”），固定防御依然失效（残留10-17%），需要可信带外交互渠道，这是一个根本性开放问题。
    - 攻击策略仅从固定池中搜索，未使用梯度或LLM循环优化，更强攻击可能存在。
- **算力不明确**：未报告训练/推理资源，可能影响可重复性的透明度。
- **重复次数有限**：每个条件3-6次重复，虽然统计检验（bootstrap）支持结论，但增加重复次数可提高置信度。
- **摘要存活评定**：依赖LLM裁判（尽管通过三裁判多数投票验证），人类标注可能更精确。
