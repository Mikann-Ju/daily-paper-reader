## 摘要
大型语言模型（LLM）正迅速演变为能够与外部工具和环境交互的代理系统，这引入了新的安全风险，例如通过不可信的外部来源进行的间接提示注入攻击。现有的防御措施主要集中在推理时阻止恶意内容，而当前的红队测试方法主要优化攻击成功率。因此，开发人员对于潜在提示注入如何在代理中产生和传播的可见性有限。我们提出PI-Hunter，一种用于在LLM代理中主动暴露漏洞的自动化代理审计框架。PI-Hunter构建真实的源感知测试用例，并通过反馈驱动的探索迭代演化它们，以诱导代理检索并揭示嵌入在外部环境中的潜在恶意指令。跨多个基准测试、代理架构、攻击和防御的广泛实验表明，PI-Hunter在强自动化红队测试基线上显著提高了漏洞暴露和攻击面覆盖率，同时在现有提示注入防御下仍然有效。

## Abstract
Large Language Models (LLMs) are rapidly evolving into agentic systems that interact with external tools and environments, introducing new security risks such as indirect prompt injection attacks through untrusted external sources. Existing defenses mainly focus on blocking malicious content at inference time, and current red-teaming methods primarily optimize attack success. As a result, developers have limited visibility into how latent prompt injections emerge and propagate through agents. We propose PI-Hunter, an automated agentic auditing framework for proactive vulnerability exposure in LLM agents. PI-Hunter constructs realistic source-aware test cases and iteratively evolves them through feedback-driven exploration to induce agents to retrieve and reveal latent malicious instructions embedded within external environments. Extensive experiments across multiple benchmarks, agent architectures, attacks, and defenses demonstrate that PI-Hunter substantially improves vulnerability exposure and attack-surface coverage over strong automated red-teaming baselines, while remaining effective under existing prompt injection defenses.

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想
- **诊断型“猎人”而非攻击型“优化器”**：通过**源感知测试用例生成**、**轨迹级评估**、**反馈驱动突变**和**补丁-重探索**，系统性地绘制代理的安全边界，暴露隐藏的注入路径。

### 2.2 技术细节与算法流程（见算法1）

1. **静态分析**  
   - 一次性分析代理可用的工具/接口（如 `get_unread_emails`、`search_web`）及可能受污染的外部源。  
   - 识别高风险的摄入通道（邮箱、网页、文档等）和可执行敏感操作的工具。

2. **进化利用循环**  
   - **源感知元播种**：对每个高风险源/接口生成专属测试用例（如“检查未读邮件”），而非通用领域级提示。每个用例包含良性用户意图、激活指定工具、要求暴露检索内容三个目标。  
   - **轨迹执行与评估**：执行测试用例，记录完整轨迹（工具调用、推理、输出）。采用**评估器**（基于规定标准：意图偏差、第三方指令存在、权威混淆、异常工具使用、敏感动作）进行轨迹级审计，输出结构化诊断反馈（而非二值标签）。  
   - **反馈驱动突变**：根据反馈，由LLM选择适用突变算子（如**通用算子**：体积扩展、参数模糊、指令层次覆盖；**领域专用算子**：如规划执行场景中的“新员工入职”、金融场景中的“欺诈调查”），对测试用例进行变异。  
   - **元突变**：LLM根据历史有效性对突变算子本身进行总结、改进和组合，实现探索策略的自适应演化。

3. **验证与协同进化**  
   - 暴露注入路径后，**修补器**对已发现路径应用轻量级临时缓解（如黑名单、限制接口），然后**重新引入**进化循环，迫使代理探索其他攻击面，逐步提高覆盖范围。

### 2.3 算法伪码（文字说明）
1. 初始化集合 V（脆弱路径）为空，初始化突变算子池 M，通过静态分析获取代理交互图 G。  
2. 基于 G 生成初始源感知种子测试用例 Q。  
3. 循环迭代 t=1..T：  
   - 执行 Q 收集轨迹 T。  
   - 评估 T 得到发现 F，提取新路径 V_new。  
   - 若 V_new 非空，修补代理（临时缓解），加入 V。  
   - 基于 F 选择父代测试用例，再经反馈突变生成新 Q。  
   - 对突变算子进行元突变。  
4. 返回全部脆弱路径 V。
