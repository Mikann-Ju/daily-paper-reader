## 摘要
随着LLM代理日益自主地选择工具，它们在具有不同权限的工具之间的选择变得与安全相关。然而，先前的工具选择研究侧重于安全无关的元数据偏好，使得特权敏感的选择未得到充分探索。为填补这一空白，我们研究了过度特权工具选择，即代理在存在足够低权限替代方案的情况下仍选择或升级到更高权限工具。我们引入了ToolPrivBench来评估代理在存在足够低权限替代方案时是否仍选择更高权限工具，同时测量初始选择以及瞬态工具故障后的升级。跨越八个领域和五种反复出现的风险模式，我们发现过度特权工具选择在主流LLM代理中普遍存在，并且瞬态故障进一步加剧了这一问题。我们还发现，一般的安全对齐无法可靠地迁移到最低权限工具选择，而提示级别的控制在瞬态故障下仅提供有限的缓解。因此，我们引入了一种特权感知的后训练防御，教导代理偏好足够的低权限工具，仅在必要时升级。我们的缓解实验表明，这种防御在保持通用能力的同时，显著减少了不必要的高权限工具使用。

## Abstract
As LLM agents increasingly select tools autonomously, their choices among tools with different privileges become safety-relevant. However, prior tool-selection studies focus on safety-agnostic metadata preferences, leaving privilege-sensitive choices underexplored. To address this gap, we study over-privileged tool selection, in which an agent selects or escalates to a higher-privilege tool despite a sufficient lower-privilege alternative. We introduce ToolPrivBench to evaluate whether agents choose higher-privilege tools despite sufficient lower-privilege alternatives, measuring both initial selection and escalation after transient tool failures. Across eight domains and five recurring risk patterns, we find that over-privileged tool selection is common among mainstream LLM agents and is further amplified by transient failures. We further find that general safety alignment does not reliably transfer to least-privilege tool choice, while prompt-level controls provide only limited mitigation under transient failures. We therefore introduce a privilege-aware post-training defense that teaches agents to prefer sufficient lower-privilege tools and escalate only when necessary. Our mitigation experiments show that this defense substantially reduces unnecessary high-privilege tool use while preserving general capabilities.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：LLM代理在自主选择工具时，倾向于选择更高权限的工具，而忽略已经足够完成任务的低权限替代品，作者将这种行为定义为“过度特权工具选择”（over-privileged tool selection）。
- **研究动机**：
  - 随着LLM代理从对话助手演变为自主执行体（如vibe coding），工具选择的自主权增加，但安全风险被忽视。
  - 实际部署中，不同工具具有不同的权限、范围、持久性或数据访问能力。代理可能选择“看上去更容易、更灵活”的高权限工具，即使低权限工具足以完成任务，从而扩大错误或被利用的潜在影响范围（blast radius）。
  - 现有研究主要关注有害输出、提示注入等安全攻击，而对工具选择中的权限偏差关注不足；工具选择偏见研究集中在元数据偏好上，未涉及权限维度；系统级权限控制研究则侧重于外部强制执行，而非代理内部行为。
- **整体含义**：过度特权工具选择是一种隐蔽但重要的安全威胁，需要专门的评估和缓解手段，常规安全对齐方法无法自动迁移到最低权限选择行为。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（文字说明）

- **核心思想**：通过构造仿真环境，控制工具的功能充分性（所有工具均可独立完成任务），从而将高权限工具的使用归因于代理的权限偏好而非能力不足。评估两种行为形式：
  - ** агрессивный выбор (aggressive selection)**：代理初始直接选择高权限工具。
  - ** преждевременная эскалация (premature escalation)**：代理在低权限工具因瞬态故障（如连接错误）失败后，切换到高权限工具，而未充分探索其他低权限替代品。
- **评估框架 ToolPrivBench**：
  - 每个场景包含6个工具：3个低权限（标准）工具 + 3个高权限（风险）工具，全部足以独立完成任务。
  - 多轮交互上限为5轮（k=5），第一轮低权限工具调用会返回瞬态错误，后续调用成功。
  - 核心指标：
    - **OPUR@k**：在k轮内，代理使用任何高权限工具的比例（且低权限替代品存在未尝试的情况）。
    - **PED (Pre-Escalation Exploration Depth)**：在首次使用高权限工具之前尝试过的不同低权限工具数量。PED=0表示 агрессивный выбор，≥1表示 преждевременная эскалация.
- **特权感知后训练防御**：
  - **训练数据构建**：生成不重叠的训练场景，包含标准工具和风险工具。标准工具在首次调用时返回瞬态错误，鼓励重试或尝试其他低权限工具。
  - **两阶段训练**：
    1. **SFT（监督微调）**：使用理想轨迹（展示推理权限、区分瞬态失败与能力限制、选择低权限工具）进行LoRA微调。
    2. **RL（强化学习，GRPO）**：从SFT模型初始化，在仿真环境中使用基于轨迹的奖励函数优化。奖励函数编码有序偏好：成功使用低权限工具完成>探索后适当升级>过早高权限工具使用受罚。
  - 奖励函数形式化（附录B.1）：
    - 若任务仅用标准工具完成，奖励+1；
    - 若尝试了所有低权限工具后才升级（合理探索），奖励+0.3；
    - 若未充分探索就使用高权限工具，奖励-1；
    - 其他情况+0.1；并附加长度惩罚clip(Δ, -0.6, 0.1)。

## 6. 论文的主要结论与发现

- **过度特权工具选择普遍存在**：11个模型中有6个OPUR超过30%，小模型（Qwen3-8B: 64.9%, LLaMA-3.1-8B: 55.9%）尤其严重。高级模型（Claude 4.6 Sonnet: 2.6%）较低但仍存在。
- **瞬态故障显著加剧升级行为**：多数模型的PED分布显示PED=0的比例较小，但PED=1或2的升级大幅增加（如GPT-5.2的零次选择仅5例，但一次失败后升级增至13例，二次失败后增至35例）。重复失败侵蚀对低权限工具的信心。
- **领域和风险类型差异明显**：基础设施领域OPUR最高，医疗和政府领域较低。Authority Escalation和Safety Bypass是最常见的风险类型，Scope Expansion最低。
- **安全对齐不迁移到最低权限选择**：AgentAlign虽能大幅降低有害行为，但对OPUR影响微弱甚至反向（Qwen2.5-7B的OPUR从50.4%升至60.7%），说明拒绝有害行为不等于学会选择最小权限。
- **提示工程缓解有限**：提示能降低OPUR，但在多轮交互（失败后）效果减弱。
- **特权感知后训练有效**：在Qwen3-4B/8B/4B-Think上，OPUR分别降至39.7%、27.0%、18.9%（基线和提示工程均远高于此），且对MMLU、GSM8K、MetaTool的保留率超过95.8%。

## 7. 优点

- **问题新颖且实用**：首次系统定义并量化LLM代理中的过度特权工具选择问题，填补了工具选择安全维度的空白。
- **基准设计严谨**：通过功能充分性约束、双重验证（自动+人工）和描述中立性检查，有效隔离了能力混淆因素，使高权限使用可归因于偏见而非能力不足。
- **评估维度全面**：同时覆盖初始选择和故障后升级，引入PED区分 агрессивный选择 和 premature 升级；跨领域和风险类型分析揭示了模式。
- **缓解方法有效且通用**：结合SFT和GRPO的后训练机制，在保持通用能力的前提下显著降低过度特权使用，且具备跨模型容量泛化性。
- **分析深入**：揭示了安全对齐不自动迁移这一反直觉现象，具有重要实践指导意义。

## 8. 不足与局限

- **实验环境为模拟**：所有评估在仿真环境中进行，未接入真实生产工具或外部服务，可能遗漏真实世界的复杂性（如网络延迟、权限错误语义等）。
- **任务规模受限**：每个场景只有6个工具、5轮交互，无法覆盖大规模工具集、部分功能重叠工具、多工具协作工作流等复杂场景。
- **工具功能完备性假设过强**：所有工具均被构造成独立完成任务，但在实际中低权限工具可能需要组合使用或存在功能局限性，单纯偏好最小权限可能不合理。
- **模型覆盖范围有限**：缓解实验仅针对Qwen系列（3个变体），未在其他模型家族（如Claude、GPT-4/5）上验证后训练的泛化性（论文提到资源限制）。
- **风险定义抽象**：五类风险类型基于人工抽象，可能无法涵盖所有特权滥用模式，且跨数据集的通用性需验证。
- **未深入分析机制**：论文指出故障导致升级加大的现象，但未深入探讨认知或不确定性层面的根本原因（如工具排序偏差、执行失败后的置信度崩溃等）。作者将此作为未来工作。
